# Delivery Example: Model Discovery

## Delivery Contract

- audience: user or project lead, not final external reader yet.
- format: interim human-readable framing memo.
- tone: plain language with enough reasoning to support a decision.
- internal_information_boundary: mention candidate frameworks and evidence needs; avoid machine-board fields and gate names.

## Human-Readable Output Shape

1. The project is not ready for evidence filling because the explanation frame is still open.
2. There are 3-5 plausible ways to understand it.
3. Each frame would require a different evidence path.
4. The next small probe will decide whether to converge or keep exploring.
5. No final external claim should be made yet.

## Machine Recovery Note

- keep_separate: yes
- include: candidate frameworks, argument map, discriminating probe, convergence conditions, next_route.
