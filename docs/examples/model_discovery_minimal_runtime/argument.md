# Argument Example: Model Discovery

## Issue

What should the project primarily be understood as?

## Positions

| Position | Pro | Con | Unresolved Question |
| --- | --- | --- | --- |
| Access Framework | Easy for third-party readers to understand; maps to social value. | May require stronger beneficiary evidence. | Who gained access, and how can we show it? |
| Workflow Framework | Often easier to support with process evidence. | May not explain why the project matters beyond efficiency. | What workflow changed in observable terms? |
| Capability Framework | Strong if the output is reusable across projects. | Can feel abstract without concrete examples. | Has anyone reused or adapted it? |
| Governance Framework | Fits Complex-style work and decision quality. | Risks overemphasizing process. | Which decision became better because of governance? |

## Current Lean

- tentative_position: none
- reason: Need one discriminating probe before convergence.

## Open Questions

- Which audience will judge the final explanation?
- What evidence type is easiest and strongest to obtain?
- Which framework would fail fastest if wrong?
