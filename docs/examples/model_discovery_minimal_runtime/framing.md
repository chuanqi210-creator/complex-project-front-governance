# Framing Example: Model Discovery

## Problem Domain

- issue: What is the best explanation path for the project?
- why_unsettled: The current materials could support multiple stories, and selecting one too early would bias evidence collection.

## Candidate Frameworks

| Framework | Core Hypothesis | Best Evidence To Distinguish | Main Risk |
| --- | --- | --- | --- |
| Access Framework | The project matters because it expands access to an important resource or capability. | Evidence that access changed for a concrete group. | May ignore quality or adoption. |
| Workflow Framework | The project matters because it improves how people work. | Before/after workflow evidence or user behavior. | May overclaim impact from process change. |
| Capability Framework | The project matters because it creates a new reusable capability. | Reuse, transfer, or independent application evidence. | May sound abstract without concrete outcomes. |
| Governance Framework | The project matters because it improves decision quality and accountability. | Clear decision records, role boundaries, or risk reduction. | May become too process-heavy for the audience. |

## Divergence Budget

- minimum_candidates: 3
- maximum_candidates: 5
- convergence_not_allowed_until:
  - at least one discriminating probe is defined
  - at least two rejected or downgraded frameworks have reasons
  - the user-facing problem statement is stable
