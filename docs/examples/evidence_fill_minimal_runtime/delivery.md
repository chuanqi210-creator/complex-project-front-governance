# Delivery Example: Evidence Fill

## Delivery Contract

- audience: third-party reader.
- format: human-readable summary plus compact evidence table.
- tone: clear, non-technical, no internal protocol jargon.
- internal_information_boundary: do not include machine-board fields, route names, or tool-state details unless they change the reader's decision.

## Human-Readable Output Shape

1. What the fixed rubric is trying to support.
2. Which metrics are well-supported.
3. Which metrics are only partly supported.
4. What cannot be claimed yet.
5. What evidence would upgrade the weak claims.

## Machine Recovery Note

- keep_separate: yes
- include: current_basis, evidence gaps, next_route, source authority decisions.
