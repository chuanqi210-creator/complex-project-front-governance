# 外部真实团队贡献署名协作压力测试：2026 大创项目申报

current_as_of：2026-06-22

## 一、本轮选择理由

本轮沿第 114 节机器看版的 `next_route`，从现实时间所有主要比赛和真实组织项目中选择“团队贡献、署名排序、成员退出、指导关系、协作冲突或项目组织责任”方向，而不是继续在 AI 科研、知识产权或上一轮主题内部迭代。

选用浙江工商大学统计与数学学院 2026 年度大学生创新创业训练计划项目申报通知作为现实样本，是因为大创项目天然包含团队人数、明确分工、指导教师、进展记录、中期检查、结题验收、项目负责人和主要内容变更限制等协作治理信号。它能暴露一个常见断裂：项目材料中有团队名单、指导教师和过程记录，但这些材料不自动证明谁真正做了关键贡献、署名或荣誉是否公平、成员退出是否无争议、导师责任是否真正履行。

## 三、暴露出的失败模式


## 四、为什么既有 trace 不够

1. `intellectual_property_license_rights_trace` 能处理成果权属和许可，但不能完整处理团队内部贡献公平、成员退出、导师指导和协作冲突。
2. `volunteer_service_operation_trace` 能处理支队运行和权益保障，但不能覆盖论文署名、竞赛荣誉分配、作者资格和共同责任。
3. `open_source_contribution_trace` 能处理 issue、review、merge 和维护交接，但不能覆盖线下团队、课程项目、大创项目和指导教师责任。
4. `student_association_governance_trace` 能处理社团治理，但不能覆盖科研/竞赛团队中“名单存在但贡献不可证”的问题。

因此需要新增 `team_contribution_authorship_collaboration_trace`，作为横跨论文、比赛、课程、支教、志愿服务、开源和企业项目的协作治理追踪。

## 六、最小结论门

| 想写的结论 | 至少需要的证据 | 证据不足时的降级表达 |
| --- | --- | --- |
| 团队贡献明确 | 角色职责、贡献标准、关键交付物、工作量和质量记录、版本或会议记录 | 已有团队分工安排，贡献证据待核验 |
| 署名或荣誉分配公平 | 事前署名/荣誉规则、贡献证据、全体成员知情同意、异议处理记录 | 已形成署名或荣誉方案，公平性仍需贡献证据支撑 |
| 指导教师有效 | 指导职责、指导频率、方案修改记录、过程反馈、风险纠偏证据 | 已配置指导教师，指导效果待过程证据验证 |
| 成员退出无争议 | 退出原因、交接材料、权益处理、后续访问权限和成员确认 | 已发生成员变更，权益与交接风险待核验 |
| 协作稳定且责任可追溯 | 决策记录、冲突处理、数据代码材料归属、错误纠正和后续维护责任 | 已有协作过程材料，责任链仍待补齐 |
