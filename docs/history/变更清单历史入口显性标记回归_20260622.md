# 变更清单历史入口显性标记回归 2026-06-22

## 1. 小题背景

上一轮 `current_scope_recovery_scan` 已能区分当前恢复入口和历史记录，但复扫变更清单时发现一个细小缺口：部分旧恢复入口虽然事实上只是历史档案，却没有显性写出“历史记录”或“不再作为当前入口”。这会让后续扫描器重新依赖上下文推断，增加误报概率。

## 3. 归纳出的共性规则

```yaml
failure_mode_id: ambiguous_archive_recovery_row
rule:
  trigger_signal: change_inventory 或 release_package 中出现旧机器看版、旧 next_route、旧恢复入口
  required_behavior:
    - 如果它是当前入口，必须出现在明确 current_scope 中
    - 如果它是历史档案，必须在同一行或同一条记录中写出历史记录、不再作为当前入口或 archived_recovery_entry
    - 如果既非当前入口又无历史标签，标为 ambiguous_archive_hit，先修复标签再继续扩展
  evidence_check:
    - old_recovery_entries_without_history_label_count == 0
  route_back:
    - 回到 change_inventory 当前恢复入口段和历史产物表，补明当前/历史身份
  friction_effect: 不增加用户提问，只提高机器恢复可读性
```

## 4. 本轮修复

1. 将变更清单中旧第 152 节恢复入口相关行改为“历史记录”或“曾更新”，并写明当前入口见第十六节。
2. 修正变更清单第十六节 stale 文本：持续优化 Codex goal 当前为 active，不再写“当前没有 active Codex goal”。
3. 在主协议 `current_scope_recovery_scan` 中加入 `ambiguous_archive_hits` 和旧入口显性标记要求。
4. 在路由表加入“旧入口显性标签”检查。
5. 在长日志追加本轮回归记录和新的当前机器看版。
