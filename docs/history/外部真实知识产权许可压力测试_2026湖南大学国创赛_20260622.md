# 外部真实知识产权许可压力测试：2026 湖南大学国创赛选拔

current_as_of：2026-06-22

## 一、样本选择

本轮选择湖南大学教务处 2026 年 3 月 30 日发布的《关于举办中国国际大学生创新大赛（2026）湖南大学选拔赛的预通知》作为压力测试样本。

选择理由：

1. 它是现实时间正在发生的高校创新创业竞赛项目，不属于本地 AI 科研项目，也不是抽象法律讨论。
2. 通知要求团队是项目实际核心成员，不得借用他人项目参赛；产业赛道要求应答材料拥有自主知识产权，不得侵犯他人知识产权或物权；参赛项目涉及发明创造、专利技术、资源等必须拥有清晰合法的知识产权或物权；推荐材料包括知识产权清单、专利证书、论文、检测报告等证明材料和团队承诺书。
3. 它能暴露一个高频失败模式：把“有专利/软著/论文/知识产权清单/承诺书/开源仓库/授权截图”误写成“权属清晰、可商用、无侵权风险、成果可转化”。
4. 它能把真实项目池继续扩展到知识产权、成果归属、版权许可、专利软著、开源许可证、数据权益、校企合作成果和商业使用边界。

## 三、暴露的失败模式

```yaml
failure_mode:
  id: ip_certificate_license_clearance_gap
  pattern: 把专利申请、授权专利、软著登记、商标证书、论文发表、代码仓库、开源许可证、素材来源声明、AIGC 生成记录、授权截图、承诺书、竞赛规则、合作协议、导师或企业口头认可误当成权属清晰、可自由使用、可商用、无侵权风险、团队贡献明确、成果可转化、开源可闭源或可再授权
  consequence:
    - 把证明材料清单误写成权属链闭环
    - 把登记或授权状态误写成自由实施和商业使用
    - 把开源可用误写成无条件可闭源、可再授权或无署名义务
    - 把团队承诺书误写成共同创作、职务成果和第三方素材争议已经解决
  corrected_behavior: 启用 intellectual_property_license_rights_trace，先核验权利客体和权利主体，再追踪形成过程、贡献记录、职务或委托成果、合作协议、第三方素材、开源依赖、数据和模型授权、许可范围、转让登记、商业使用、FTO 检索、收益分配和争议处理；证据不足时降级为已提交或登记的知识产权材料、待核验的授权许可假设、待清理的第三方权利风险
```

## 四、为什么既有 trace 不够

1. `business_model_validation_trace` 能处理创业项目市场验证，但不能证明权属链、许可证兼容、职务成果和第三方素材清理。
2. `enterprise_delivery_trace` 能处理客户验收和上线交付，但不能说明代码、数据、模型、专利、商标、素材和论文成果的权利边界。
3. `open_source_contribution_trace` 能处理开源 PR、许可证和上游合并，但不能覆盖竞赛成果、专利软著、技术转让、校企合作和商业授权全链条。
4. 因此需要新增 `intellectual_property_license_rights_trace`。

## 六、最小结论门

1. 证明“竞赛要求知识产权材料”：可以引用湖南大学通知中的自主知识产权、不得侵犯他人知识产权或物权、知识产权清单和证明材料要求。
2. 证明“某项目提交了知识产权证明”：必须看到该项目的知识产权清单、证书、论文、检测报告、授权材料和承诺书。
3. 证明“权属清晰”：必须有权利客体清单、权利主体、职务/委托/合作边界、贡献记录、团队内部约定和第三方材料授权。
4. 证明“可商用或可转化”：必须有许可或转让合同、商业使用范围、开源依赖兼容、专利实施自由、保密和收益分配安排。
5. 证明“无侵权风险”：不能绝对化，只能在检索范围内说明已做 FTO/版权/商标/开源/数据授权核验和残余风险。
