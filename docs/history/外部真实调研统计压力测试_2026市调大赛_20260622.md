## 二、为什么选择这个样本

前几轮已覆盖 AI 数据赛、公益实践、创业、硬件、开源、数值计算、数学建模和传播设计，但还缺少一类和社会实践最贴近的项目：问卷、访谈、市场调查、社会调查和统计报告。

这类项目在支教、县域调研、公共议题提炼和社会实践汇报中非常关键。它的典型风险是：

1. 样本数量多，不等于样本具有代表性。
2. 图表漂亮，不等于变量定义、缺失值、抽样方式和测量误差可靠。
3. 访谈故事动人，不等于能代表群体经验。
4. 显著性检验通过，不等于有因果关系或政策建议成立。
5. 报告获奖，不等于调研伦理、隐私保护和受访者权益已处理。

## 三、本轮暴露的新协议缺口

现有 `service_impact_trace` 能处理服务影响，`model_validity_trace` 能处理模型到现实决策的转译，但还缺少一个直接处理“调研证据是否能支撑社会结论”的追踪。

例如一支支教队在县中做 AI 认知问卷：

1. 收到 300 份问卷，访谈 12 名学生。
2. 发现多数学生对 AI 焦虑，少数学生已在使用 AI 辅助学习。
3. 汇报时写成“县中学生普遍缺乏 AI 时代路径认知”。

如果没有说明抽样框、年级比例、班级选择方式、非回应偏差、问题诱导性、访谈同意、隐私保护、数据清洗、定性编码和结论外推边界，就不能声称“县中学生普遍如何”。更稳妥的结论只能是“在本次接触到的样本中出现了某些值得继续验证的认知差异和路径困惑”。

## 五、最小结论门

调研统计类项目只有同时满足以下条件，才能把样本结果上升为较强社会结论：

1. 研究问题清楚，知道自己要回答“是什么、为什么、如何变化、是否有效”中的哪一种。
2. 目标总体清楚，不能把方便样本直接写成总体。
3. 抽样方式、样本结构、非回应偏差和选择偏差已说明。
4. 问卷或访谈提纲经检查，不存在明显诱导、双重问题或概念混用。
5. 涉及未成年人、弱势群体、学校场景或敏感信息时，有知情同意、匿名化、最小采集和安全存储安排。
6. 数据清洗、缺失值、异常值、定性编码和统计方法可复核。
7. 明确区分相关、解释、预测和因果。

如果上述条件不足，最终报告必须降级：

| 不能写 | 应降级为 |
| --- | --- |
| 县中学生普遍存在某问题 | 本次样本中观察到某问题线索 |
| 问卷证明某因素导致某结果 | 问卷显示二者相关，因果机制待验证 |
| 访谈表明学生真实想法就是如此 | 访谈呈现若干个体经验，需结合更多样本和情境解释 |
| 调研报告可直接给出政策建议 | 当前调研可提出待验证的问题假设和初步建议 |

## 六、对前置协议的修正

新增失败模式：

```yaml
agent_failure_modes:
  - failure_id: F45-01
    name: survey_truth_gap
    symptom: 把问卷数量、访谈故事、统计图表、显著性检验或调研报告获奖误当成真实社会结论、群体代表性或因果解释成立
    root_cause: 只追踪调研产出，没有追踪抽样框、样本偏差、测量效度、知情同意、隐私保护、数据清洗、统计边界和结论外推
    trigger_signal:
      - 问卷、访谈、市场调查、社会调查、用户研究、调研报告或统计分析项目
      - 结论中出现“普遍、证明、导致、代表、真实反映、应当推广”等强表述
      - 涉及未成年人、学校、县域、弱势群体、敏感数据或公共政策建议
    corrected_behavior: 启用 survey_research_validity_trace，并把调研结论降级到证据能够支持的范围
    detection_check: 桌面协议应能检索到总规则 42、执行闸门 101、survey_research_validity_trace 和 survey_truth_gap
    regression_case: R45-survey-research-validity-trace
    status: mitigated_with_trace_module
```

新增回归样例：

```yaml
agent_regression_cases:
  - case_id: R45-survey-research-validity-trace
    user_prompt_pattern: 用户要求做问卷、访谈、市场调查、社会调查、用户研究、公共议题调研或统计报告，并希望得出群体结论或解决方案
    expected_behavior: 先核验真实项目来源或研究场景，再启用 survey_research_validity_trace，区分样本发现、代表性、解释边界、伦理边界和可行动建议
    forbidden_behavior:
      - 只给问卷题目和报告目录
      - 把样本数量、显著性、图表或访谈金句当成总体事实
      - 不说明抽样、非回应、知情同意、隐私保护和因果边界
    required_tools_or_checks:
      - live_project_pool_scan
      - source_link_recency_audit
      - trace_module_router
      - survey_research_validity_trace
    pass_condition: 最终表达中能清楚看到本次调研能证明什么、不能证明什么、需要什么补充证据才能升级结论
    last_tested: 2026-06-22
```

## 七、对用户低摩擦入口的影响

用户不需要说“survey_research_validity_trace”。只要出现以下自然语言，AI 应自动触发：

1. “问卷、访谈、调研、市场调查、社会调查、用户研究、统计报告、样本、显著性、相关性、群体结论”。
2. “我们想了解学生/用户/公众/某地群体真实想法”。
3. “基于调研给出政策建议、公共议题、产品建议、课程改进或组织方案”。

用户可见回复只需要说明：

“这是调研/问卷/访谈/统计报告类项目，我会额外检查抽样、测量、知情同意、隐私、统计边界和结论外推，避免把样本发现误写成总体事实。”
