# 外部真实应急安全压力测试：2026 全国安全生产月

current_as_of：2026-06-22

## 一、样本选择

本轮选择国务院安委会办公室、应急管理部 2026 年 5 月 20 日发布的《关于开展2026年全国“安全生产月”活动的通知》作为压力测试样本。

选择理由：

1. 它是现实时间正在发生的国家级安全生产和应急管理项目，不属于本地 AI 科研项目，也不是普通竞赛项目。
2. 它覆盖安全宣传、风险隐患排查、重大事故隐患标准、重点行业领域、明查暗访、媒体曝光、数字推演、实战模拟、综合演练和安全宣传咨询日。
3. 它能暴露一个高频失败模式：把通知、宣传、培训、预案、演练照片、隐患清单或某次未发生事故误写成风险已经降低、隐患已经消除、应急能力已经形成。
4. 它能把真实项目池继续扩展到应急管理、安全生产、事故处置、大型活动风险预案、消防疏散和人员密集场所安全治理。

## 三、暴露的失败模式

```yaml
failure_mode:
  id: safety_notice_drill_outcome_gap
  pattern: 把安全生产月通知、安全宣传、活动方案、风险评估报告、隐患清单、应急预案文本、培训签到、演练照片、安全承诺书、安全许可、媒体报道、现场热闹或某次未发生事故误当成风险已降低、隐患已消除、应急能力提升、安全可控、活动安全成功或事故不会发生
  consequence:
    - 把安全治理动员误写成风险治理结果
    - 把预案文本误写成现场处置能力
    - 把隐患发现误写成隐患消除
    - 把一次未出事误写成长期安全可控
  corrected_behavior: 启用 emergency_safety_risk_response_trace，先核验适用法律政策和项目场景，再追踪风险源、暴露人群或流程边界、责任主体、许可审批、安全工作方案、重大事故隐患标准、隐患台账、整改验收、培训演练评估、现场控制、疏散救援、消防医疗通信资源、事故报告调查、复盘追责和残余风险；证据不足时降级为安全宣传或预案材料、待执行的隐患排查整改、待验证的应急能力假设
```

## 四、为什么既有 trace 不够

1. `event_hosting_operation_trace` 能处理大型活动承办是否公平有序，但不能充分追踪安全生产风险源、重大事故隐患判定、整改验收和事故报告调查。
2. `security_remediation_trace` 处理网络安全漏洞闭环，不覆盖生产安全、消防疏散、人员密集场所、危险化学品、矿山、燃气或大型活动现场风险。
3. `labor_education_practice_trace` 能要求劳动实践安全培训，但不能覆盖安全生产月、隐患排查、应急预案、事故处置和复盘追责的全链条。
4. 因此需要新增 `emergency_safety_risk_response_trace`。

## 六、最小结论门

1. 证明“全国安全生产月通知存在”：可以引用应急管理部通知标题、发布时间、主题、6 月 16 日安全宣传咨询日和合肥主场活动。
2. 证明“安全治理要求存在”：可以引用通知中的隐患举报、重大事故隐患判定标准、重点行业领域、明查暗访、媒体曝光、数字推演、实战模拟和综合演练。
3. 证明“某个单位或活动做了安全工作”：必须补充本单位活动方案、培训签到、演练记录、隐患台账或安全工作方案。
4. 证明“风险降低或隐患消除”：必须有风险基线、隐患分级、整改责任人、整改期限、复查验收、销号记录和残余风险。
5. 证明“应急能力提升”：必须有演练评价、问题清单、改进记录、现场资源和指挥链验证，不能只看演练照片和宣传报道。

1. 主协议新增规则 65、执行前闸门 124、`emergency_safety_risk_response_trace` 和 `safety_notice_drill_outcome_gap`。

3. 路由表新增应急安全风险治理触发条件。
4. 低摩擦入口新增自然语言触发规则。
5. 长期日志新增回归样例 `R68-emergency-safety-risk-response-trace`。
