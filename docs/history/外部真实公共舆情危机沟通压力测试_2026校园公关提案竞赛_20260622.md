## 二、暴露出的协议缺口

旧协议已有 `communication_effect_trace`，但它更适合传播触达和效果；已有 `science_communication_quality_trace`，但它更适合科普准确性；已有 `emergency_safety_risk_response_trace`，但它更适合安全风险和事故处置。公共舆情和危机沟通需要独立追踪“事实、责任、公众、回应、救济、整改和信任修复”。

如果没有这一层，AI 很容易写出这些不可靠结论：

1. “声明已经发布，所以事实已经澄清。”
2. “热搜下降，所以危机已经解除。”
3. “公关方案获奖，所以信任修复路径有效。”
4. “媒体报道正面，所以组织声誉已经恢复。”
5. “无进一步负面舆情，所以公众已经接受。”
6. “道歉和整改承诺已经发出，所以责任闭环。”

这些判断的共同问题是把传播动作当成问题解决，把短期声量当成信任修复，把组织自述当成事实核验。

## 五、判断规则

1. 声明、道歉、辟谣、新闻发布会、热搜下降、媒体报道和舆情监测截图只能说明“发生了沟通动作”，不能直接证明事实清楚、责任落实或信任修复。
2. 公关方案、创意提案、路演获奖和传播热度不能替代受影响群体的反馈、救济和长期信任指标。
3. 舆情降温可能来自平台热度衰减、删除控评、注意力转移、信息不对称、受影响者沉默或无法发声；不得把热度下降写成问题解决。
4. 政务或组织新媒体发布要追踪发布主体、授权链、内容一致性、互动反馈和后续纠错，不能只看渠道数量。
5. 涉及事故安全时同步看 `emergency_safety_risk_response_trace`；涉及服务对象或可识别个人经历时同步看 `stakeholder_consent_participant_protection_trace`；涉及个人信息或平台算法时同步看 `data_algorithm_privacy_governance_trace`；涉及科学辟谣时同步看 `science_communication_quality_trace`。

## 六、迁移到真实项目的用法

| 项目类型 | 常见误判 | 最小补证 |
| --- | --- | --- |
| 公关策划竞赛 | 提案获奖等于现实信任修复有效 | 真实事件边界、受众基线、执行证据、反馈与指标 |
| 校园危机回应 | 学校声明等于事实查清和学生接受 | 事实核验、学生反馈、申诉通道、整改验证 |
| 品牌危机 | 道歉和降热搜等于声誉恢复 | 受影响消费者反馈、补偿、复购或信任指标 |
| 政务舆情 | 官方通报等于公众理解和风险解除 | 回应时限、信息更新、问责整改、公众疑点回应 |
| 事故通报 | 发布原因和处置进展等于安全闭环 | 风险源消除、责任追究、复盘预防、第三方验证 |
| 辟谣澄清 | 辟谣文本等于误解纠正 | 谣言传播路径、证据透明度、受众理解和后续监测 |

## 九、最小结论门

没有以下材料之一时，不得写“事实已澄清、危机已解除、信任已修复、声誉已恢复、责任已落实、整改有效、风险已消除”：

1. 事件边界、事实核验、未知和争议点。
2. 受影响群体及其实际反馈。
3. 回应主体、授权链、责任边界和更新时间线。
4. 隐私、法律、安全、未成年人或弱势群体保护边界。
5. 谣言纠偏证据和传播路径。
6. 投诉申诉、救济补偿、支持和纠错机制。
7. 整改责任人、期限、资源和第三方或外部验证。
8. 信任或声誉基线、后续指标和平台传播偏差说明。
9. 复盘预防、制度修正和长期跟踪。

证据不足时使用替代表达：

1. “已完成某轮公开回应。”
2. “当前只能证明进行了危机沟通动作。”
3. “信任修复、责任闭环和整改效果仍需补充证据。”
4. “该结论不能外推为公众已经接受、组织声誉已经恢复或风险已经消除。”
