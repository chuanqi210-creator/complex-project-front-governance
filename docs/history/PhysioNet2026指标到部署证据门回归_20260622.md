# PhysioNet 2026 指标到部署证据门回归 2026-06-22

## 一、本轮真实项目样本

样本：George B. Moody PhysioNet Challenge 2026，“Screening for Cognitive Impairment During Sleep Studies”。

来源：

1. PhysioNet Challenge 2026 官网：https://moody-challenge.physionet.org/2026/
2. FDA Artificial Intelligence in Software as a Medical Device：https://www.fda.gov/medical-devices/software-medical-device-samd/artificial-intelligence-software-medical-device
3. FDA Good Machine Learning Practice for Medical Device Development：https://www.fda.gov/medical-devices/software-medical-device-samd/good-machine-learning-practice-medical-device-development-guiding-principles
4. FDA Clinical Decision Support Software guidance page：https://www.fda.gov/regulatory-information/search-fda-guidance-documents/clinical-decision-support-software

## 二、小题目标

第 188 节新增 `claim_lane_binding`，防止把活动量或产出外推为公共影响。本轮选择一个研究/数据/模型类、高风险医疗语境项目，检验另一个常见外推：把挑战赛隐藏测试集成绩、论文指标或 leaderboard ranking 直接写成“可临床部署”“能改善诊疗/决策”。

## 三、事实摘录

```yaml
source_role_map:
  physionet_2026_overview:
    role: challenge_objective_and_research_task
    can_support_claims:
      - 该挑战邀请团队用 PSG 和基础人口学数据预测未来认知障碍诊断
      - 获胜依据是隐藏测试集最高分
      - 数据来自多个美国机构并经过去标识化
    cannot_support_claims:
      - 参赛算法已经获得临床许可
      - 参赛算法已经改善患者结局
      - 隐藏测试集最高分等于临床可部署
  physionet_data_split_and_scoring:
    role: benchmark_evaluation_contract
    can_support_claims:
      - 训练集、验证集和测试集分离，验证集和测试集隐藏
      - 验证/测试来源与训练来源不同，测试集用于赛后评估
      - 评分使用挑战定义的 reward、age-conditioned AUROC 等指标
    cannot_support_claims:
      - 所有真实医院、设备、人群和工作流均可泛化
      - 模型在临床流程中安全有效
  fda_ai_samd:
    role: deployment_and_lifecycle_context
    can_support_claims:
      - 医疗 AI/ML 软件的开发、部署、使用和维护需要产品生命周期管理
      - FDA 会按适当路径审查医疗设备，软件修改也可能需要审查
    cannot_support_claims:
      - 某个挑战赛算法已经满足监管要求
  fda_gmlp:
    role: safe_effective_high_quality_medical_ai_principle_source
    can_support_claims:
      - GMLP 旨在促进安全、有效、高质量医疗 AI/ML 设备，并考虑总产品生命周期
    cannot_support_claims:
      - 单一 leaderboard 分数足以证明安全有效
  fda_cds_guidance:
    role: clinical_decision_support_regulatory_boundary
    can_support_claims:
      - FDA 对 CDS 软件是否属于 device 有边界说明
      - 面向患者/照护者或符合 device 定义的软件仍受相关数字健康政策影响
    cannot_support_claims:
      - 研究模型可绕过临床决策支持、监管或可解释性要求
```

## 四、并行证据泳道小题

```yaml
micro_task_execution_check:
  task: 用 PhysioNet 2026 构造 benchmark -> deployment 证据链，检查 claim_lane_binding 是否需要部署证据门
  observed_result:
    - benchmark lane 能证明挑战任务、数据划分、隐藏验证/测试和评分函数
    - generalization lane 能证明验证/测试来源与训练不同，但不能证明所有真实场景外部有效性
    - deployment lane 需要 FDA/临床/工作流/安全监控等额外证据，不能由测试集分数替代
  exposed_gap: benchmark_score_without_deployment_readiness_gate_can_overclaim_clinical_use
  pass_fail: pass_after_rule_tightening
```

```yaml
claim_lane_binding:
  primary_claim: algorithm_can_screen_future_cognitive_impairment_from_sleep_studies
  evidence_lanes:
    - lane_id: challenge_task_and_data_lane
      claim_ladder_level: output_or_research_task
      can_support_claims:
        - 挑战赛任务存在
        - 数据和标签定义存在
        - 训练/验证/测试结构存在
      cannot_support_claims:
        - 算法临床可用
        - 患者结局改善
    - lane_id: benchmark_score_lane
      claim_ladder_level: benchmark_quality
      can_support_claims:
        - 在挑战定义指标和隐藏测试集上的相对表现
        - 符合挑战提交和评分规则
      cannot_support_claims:
        - 真实医院部署效果
        - 医生/患者使用安全性
        - 监管可接受性
    - lane_id: deployment_readiness_lane
      claim_ladder_level: real_world_use_or_deployment
      deployment_readiness_gate:
        external_validity_evidence: 多机构/多设备/多群体外部验证或前瞻验证
        workflow_or_user_evidence: 临床工作流、医生可理解性、用户训练和使用场景证据
        risk_safety_permission_evidence: 错误代价、偏倚、公平、隐私、告知与授权边界
        regulatory_or_policy_boundary: FDA CDS/SaMD 或相应地区监管路径判断
        monitoring_and_handoff_plan: 上线监测、漂移管理、责任归属和撤回机制
        downgrade_if_missing: 只能表述为研究/挑战赛表现，不能表述为可临床部署或改善诊疗
```

## 五、规则晋升判断

```yaml
rule_promotion_mechanism:
  candidate_rule: deployment_readiness_gate
  promotion_scope: main_protocol_tightening_rule_under_claim_lane_binding
  evidence:
    - PhysioNet 2026 官方页面明确挑战获胜依据为隐藏测试集最高分
    - PhysioNet 2026 数据和测试来源可支撑 benchmark 外部性的一部分，但不等于真实部署
    - FDA AI/SaMD 和 GMLP 页面强调医疗 AI 的部署、使用、维护、风险和产品生命周期语境
  anti_protocol_bloat_gate:
    trigger_signal: 结论从 benchmark/demo/论文指标/内部验收上升为真实使用或部署
    observable_behavior: 必须填写 external_validity、workflow_or_user、risk_safety_permission、regulatory_or_policy_boundary、monitoring_and_handoff
    check: 缺少 deployment_readiness_gate 时 verifier preset 红灯
    route_back: 缺任一关键证据时降级为“在给定测试条件下表现”或“具备进一步验证潜力”
    user_friction_impact: 不增加默认提问，只在用户要求部署/实际效果主张时加深证据门
  decision: adopt_as_generic_tightening_rule
```

## 六、不能外推

本回归不评价任何 PhysioNet 2026 参赛队伍或算法，也不判断 FDA 是否会监管某个具体模型。它只证明一条通用方法：benchmark、排行榜、论文指标、离线测试和样机验收不能自动证明真实部署；当结论进入真实使用/高风险决策，必须另过 `deployment_readiness_gate`。
