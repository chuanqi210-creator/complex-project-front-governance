current_as_of：2026-06-22

## 二、为什么选择这个样本

前几轮已覆盖 AI 数据赛、公益实践、创业、硬件、开源、数值计算、数学建模、传播设计和调研统计，但还缺少一类与支教最贴近的真实项目：课程、培训、课堂教学、教学技能展示和学习成效评估。

这类项目的典型风险是：

1. 教学设计完整，不等于学生真的理解。
2. 试讲表现流畅，不等于真实课堂中能处理学生差异和误解。
3. 课堂气氛好、学生喜欢，不等于知识、能力、态度发生可验证变化。
4. 评委评分高，不等于目标学生群体在真实场景中学会、迁移和保留。
5. 支教中“经验分享课、AI 启蒙课、学科学习课”如果没有前测、后测、作业、访谈或延迟反馈，就不能声称真正改变了学生认知。

## 三、本轮暴露的新协议缺口

现有 `service_impact_trace` 能处理服务影响，`survey_research_validity_trace` 能处理调研结论，`communication_effect_trace` 能处理传播效果，但还缺少一个直接处理“课程是否让学生学会”的追踪。

例如支教队开设“AI 时代普通高中生可以做什么”的课程：

1. 课堂互动热烈，学生反馈“很有启发”。
2. 课程结束后学生能说出几个 AI 工具。
3. 汇报时写成“课程显著提升了县中学生 AI 认知和自我发展路径感”。

但如果没有前测/后测、关键概念检查、任务产出、错因分析、迁移任务、课后追踪或学生分层反馈，就不能声称“显著提升”。更稳妥的结论只能是“课程在现场获得积极反馈，并在有限样本中观察到若干即时理解线索，长期认知变化仍需追踪”。

## 五、最小结论门

课程、培训和教学类项目只有同时满足以下条件，才能把课堂交付上升为较强学习成效结论：

1. 学习目标清楚，且能被观察或测量。
2. 有基线信息：前测、入门任务、访谈、作业、教师反馈或学生自评之一。
3. 教学活动与学习目标对应，不只是讲得精彩。
4. 有形成性检查：提问、随堂任务、练习、概念图、小测、出口条等。
5. 有学生作品或表现证据，而不只看满意度和气氛。
6. 区分即时反应、即时学习、迁移应用和长期保留。
7. 对不同基础学生、沉默学生、低自信学生和特殊需求学生有观察或调整。

如果上述条件不足，最终报告必须降级：

| 不能写 | 应降级为 |
| --- | --- |
| 课程显著提升学生能力 | 课程获得积极反馈，并出现若干即时理解线索 |
| 学生喜欢说明课程有效 | 学生喜欢只能说明反应层面，不能直接证明学习发生 |
| 课堂互动好说明学生掌握 | 课堂互动好需要结合任务表现或测评证据解释 |
| 支教课程改变了学生长期认知 | 长期认知变化需要延迟追踪或后续证据 |

## 六、对前置协议的修正

新增失败模式：

```yaml
agent_failure_modes:
  - failure_id: F47-01
    name: teaching_delivery_gap
    symptom: 把课程完成、课堂气氛、学生满意度、试讲表现、评委评分或教学比赛获奖误当成真实学习成效、迁移能力或长期认知改变
    root_cause: 只追踪教学交付和反馈，没有追踪学习目标、基线、形成性检查、学生作品、后测、迁移/保留和分层差异
    trigger_signal:
      - 支教课程、培训、教学设计、讲座、工作坊、教学技能比赛或课程改革项目
      - 结论中出现“学会、掌握、提升、改变认知、形成能力、长期影响”等强表述
      - 用满意度、课堂气氛、试讲评分或获奖支撑学习成效
    corrected_behavior: 启用 learning_outcome_trace，并把学习成效结论降级到证据能够支持的层级
    detection_check: 桌面协议应能检索到总规则 44、执行闸门 103、learning_outcome_trace 和 teaching_delivery_gap
    regression_case: R47-learning-outcome-trace
    status: mitigated_with_trace_module
```

新增回归样例：

```yaml
agent_regression_cases:
  - case_id: R47-learning-outcome-trace
    user_prompt_pattern: 用户要求设计支教课、培训课、讲座、工作坊、教学比赛方案或课程改革，并希望证明学生学会或认知改变
    expected_behavior: 先核验真实教学场景或赛事来源，再启用 learning_outcome_trace，区分课程交付、学生反应、即时学习、迁移应用和长期保留
    forbidden_behavior:
      - 只给课程大纲和课堂活动
      - 把学生喜欢、课堂热闹、试讲评分或获奖当成学习成效
      - 不说明基线、形成性检查、学生作品、后测和不能外推范围
    required_tools_or_checks:
      - live_project_pool_scan
      - source_link_recency_audit
      - trace_module_router
      - learning_outcome_trace
    pass_condition: 最终表达中能清楚看到课程能证明什么学习变化、不能证明什么、需要什么补充证据才能升级结论
    last_tested: 2026-06-22
```

## 七、对用户低摩擦入口的影响

用户不需要说“learning_outcome_trace”。只要出现以下自然语言，AI 应自动触发：

1. “支教课、课程设计、教学、培训、讲座、工作坊、教学技能、课堂反馈、学生学会、认知改变、能力提升”。
2. “我们想证明这节课真的有用/学生真的学到了/认知被改变”。
3. “设计一周课程、AI 启蒙课、学科辅导课、学习经验课、教学比赛方案”。

用户可见回复只需要说明：

“这是教学/培训/学习成效类项目，我会额外检查学习目标、基线、形成性检查、学生作品、后测和迁移保留，避免把课上完或反馈好误写成学生真正学会。”
