# 外部真实程序公平压力测试：2026 清华大学接收推荐免试研究生

current_as_of：2026-06-22

## 一、选择理由

本轮沿第 124 节 `next_route`，从现实时间所有主要比赛和真实组织项目中选择“招生录取、推免推荐、荣誉名额、岗位遴选、竞赛评审、专家评分、资格审核、回避制度、申诉复议和程序正义”方向，避免继续在 AI 科研、资助公平或上一轮主题内部迭代。

本轮样本选择清华大学 2026 年接收推荐免试研究生办法。原因是推免接收同时包含申请、材料审核、综合考核、初步录取、系统确认、资格复审、信息查询和申诉等高风险节点。它比一般“评奖公示”更能暴露程序公平的主链：规则是否提前公开、候选池如何形成、谁有资格进入评审、评审专家如何组成和回避、成绩如何合成、名单如何公示、异议如何复核。

## 三、暴露的协议缺口

旧协议已有 `quality_acceptance_evaluation_trace`，能处理评分和验收质量；也有 `team_contribution_authorship_collaboration_trace`，能处理团队贡献与署名公平；还有 `financial_aid_resource_allocation_fairness_trace`，能处理资源分配公平。但招生、推免、岗位、荣誉和竞赛评审的核心并不只是“分数是否高”或“名额是否分出去”，而是：

1. 规则是否提前公开，候选人是否能获得同等信息。
2. 候选池如何形成，谁被排除在申请或推荐之外。
3. 资格条件、材料要求和真实性核验是否一致。
4. 评审组织是否独立，评委是否回避利益冲突。
5. 评分标准、权重、阈值和成绩合成是否可追踪。
6. 同分、递补、特殊政策和例外处理是否有规则。
7. 公示是否保护隐私，候选人是否有知情和申诉渠道。
8. 复核、监督、纠错和取消资格是否形成闭环。

因此新增 `admission_recommendation_selection_procedural_fairness_trace`，把“分数/名单/系统确认/专家意见”与“程序公平、择优录取、机会平等、结果可信”之间的证据链拆开。

## 六、判断规则

1. 分数、排名、入围名单、拟录取名单、推荐名单、评委意见、系统确认、公示截图、录取通知、领导签字或“择优”表述，不自动等于程序公平、专家独立、机会平等、结果可信或申诉闭环。
2. 写“公平公正、择优录取、程序透明、专家评审独立、机会平等、结果可信、无异议”前，必须说明规则依据、名额来源、候选池形成、通知可及性、资格条件、材料审核、评审阶段、评委组成、回避保密、评分标准、成绩合成、阈值、同分递补、公示范围、异议申诉、监督复核和结果确认。
3. 评审分数、专家意见和面试表现是遴选证据，不是程序正义本身；如果缺少评分 rubric、评委回避、过程记录、申诉通道或复核责任，最终结论必须降级。
4. 招生、推免、竞赛、岗位和荣誉遴选常与团队贡献、质量评价、数据隐私、资助公平和知识产权重叠；若主张是“谁应获得资源”，同时看 `financial_aid_resource_allocation_fairness_trace`；若主张是“作品质量好”，同时看 `quality_acceptance_evaluation_trace`。
5. 缺少候选池、资格核验、评分标准、回避记录、公示隐私、异议复核、递补规则或过程留痕时，最终表述必须降级为“已完成某轮评审或录取流程”“待核验的程序公平假设”“待补齐的申诉复核风险”“不能证明机会平等或专家独立的结果材料”。

## 十、最小结论门

1. 只有通知和办法时，只能说“该遴选流程已发布规则和入口”。
2. 有资格审核、评审安排和评分标准时，可以说“具备基本评审程序证据”，但不能证明实际评审无偏。
3. 有评委回避、原始评分、过程记录、公示、异议处理和复核记录时，可以谨慎说“具备程序公平闭环证据”，仍需说明样本、阶段和适用范围。
4. 要证明“公平公正、机会平等、择优录取”，还需要候选池覆盖、通知可及性、特殊政策执行、未入选者反馈、监督复核和后续纠错证据。
