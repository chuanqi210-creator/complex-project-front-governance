## 3. 暴露的失败模式

```yaml
failure_mode:
  failure_id: meta_tooling_loop_drift
  symptom: 连续自我优化在恢复链已经干净后，仍继续默认优化工具、机器看版和验证器
  root_cause: continuous cycle 只要求每轮处理一个最高杠杆问题，但缺少“恢复链已收口后回到真实项目/交付可用性”的 route-back
  trigger_signal:
    - 最近 6 个以上连续机器看版属于工具、恢复链、子代理生命周期或验证器优化
    - verify_governance_recovery_tool failure_count 为 0
    - 没有新的 stale、pending、manual_action_required 或 QA 误写
  corrected_behavior:
    - 下一轮默认选择真实项目压力测试、新领域治理构造器或最终交付可用性检查
    - 只有发现新的恢复链失败、工具验证失败或用户明确要求继续修工具时，才继续工具层优化
  detection_check:
    - recent_tool_or_recovery_streak >= 6
    - latest_recovery_failure_count == 0
    - next_route 指向 real_project_pressure_test 或 deliverable_usability_check
```
