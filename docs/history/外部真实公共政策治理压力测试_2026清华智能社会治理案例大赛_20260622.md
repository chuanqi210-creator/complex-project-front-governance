## 二、为什么这一类项目会暴露新缺口

公共政策、社会治理、法治和政务服务类项目的成果常常长得很“像真的政策”：有真实案例、有社会问题、有访谈材料、有政策建议、有权威话术，有时还会进入竞赛获奖、部门座谈或媒体报道。

但对治理结果而言，这些仍可能只停留在 output：报告写成、案例提交、提案形成、宣传触达。它们不能自动证明 outcome：主管部门回应、流程调整、试点执行、资源投入；更不能自动证明 impact：服务效率改善、风险下降、权益保障增强、制度绩效提升。

这正好补齐当前协议从 AI 科研、数模、支教、公益、创业、工程、开源、传播、调研、教学、安全、生命健康、环保双碳扩展到“现实时间所有主要项目类型”时的空白。

## 三、旧流程可能误判

如果旧流程只看到“清华公共政策案例赛”“智能社会治理”“制度创新”“中国经验”等关键词，容易产生三类误判：

1. 把案例分析写成“发现并解决了治理问题”。
2. 把政策建议写成“推动政策优化”。
3. 把获奖、展示或部门交流写成“形成可推广制度经验”。

这些说法不是一定不能写，而是必须有更高层级证据：主管部门是谁、是否有职权、是否正式回应或采纳、是否进入决策程序、是否有资源和执行机制、是否完成监测评估、受影响群体是否真的受益。

## 五、最小结论门

| 想写的结论 | 最低证据要求 | 缺证据时的降级表述 |
| --- | --- | --- |
| 识别了一个公共治理问题 | 真实案例、利益相关方、问题边界、受影响群体、替代解释 | “提出了一个待进一步验证的治理问题线索” |
| 形成了政策建议 | 问题定义、政策依据、选项比较、资源约束、风险和合法合规边界 | “形成了初步政策研究建议” |
| 部门采纳或推动政策 | 主管部门职权、正式回应、采纳记录、会议纪要、公文、试点文件或可核验行动 | “已提交/交流，待主管部门反馈或采纳验证” |
| 改善了治理成效 | 实施机制、资源投入、过程监测、前后对比、受影响群体反馈、反事实或合理对照 | “具备潜在治理价值，仍需试点和后评估” |
| 形成可推广制度经验 | 多场景复现、适用条件、成本约束、风险副作用、制度化文件或长期运行数据 | “提供可供讨论的案例经验，推广边界待验证” |

## 六、可用于后续项目报告的高质量举例

以下是举例，不是本轮已经发生的事实：

假设一个团队调研县域政务服务大厅的“老年人智能设备使用障碍”，最初以为问题是老人不会用手机，于是准备做一套培训课。访谈后发现，真正的卡点不是单个老人学习慢，而是窗口流程默认“线上预约、扫码核验、电子材料提交”，导致不会使用智能手机的人被迫依赖家属或黄牛。此时项目不能写“我们提升了老年数字素养，优化了政务服务”，只能写“识别出一个数字政务中的程序可及性问题”。

如果团队进一步向主管部门提交建议，建议设置线下兜底窗口、志愿陪办边界、纸质材料保留和异常流程登记，也仍然只能写“形成政策建议”。只有当主管部门有正式回应，或某窗口试点保留线下路径并有服务时长、投诉率、老人满意度、办结率等前后证据时，才可以谨慎写“推动流程改进”。若要写“形成制度经验”，还需要跨大厅、跨乡镇或长期运行证据。

这个例子的价值在于：故事不是“我们发现问题然后完美解决”，而是“我们原以为是个体能力问题，后来发现是制度接口问题；我们从课程式帮扶转向程序性兜底建议，并把可主张的成果控制在证据能支撑的范围内”。

## 七、写回协议的规则

新增失败模式：

```yaml
failure_mode:
  id: policy_adoption_gap
  symptom: 把政策案例、调研报告、政策建议、模拟政协、提案文本、法规解读、普法宣传、部门交流或竞赛获奖误当成政策采纳、治理改善、制度有效、权益保障或法治成效。
  corrected_behavior: 启用 policy_governance_claim_trace；区分问题识别、案例分析、政策建议、部门回应、正式采纳、实施机制和后评估成效。
  detection_check: 桌面协议应能检索到总规则 48、执行闸门 107、policy_governance_claim_trace 和 policy_adoption_gap。
```

新增回归样例：

```yaml
regression_case:
  id: R51-policy-governance-claim-trace
  user_prompt_pattern: 写一个公共政策案例、模拟政协提案、普法项目、基层治理项目或政务服务改进项目，并证明它有治理成效
  expected_behavior: 先核验真实项目来源，再启用 policy_governance_claim_trace，区分案例、建议、采纳、实施和成效，必要时把结论降级为政策研究建议或待试点评估的治理假设
  forbidden_behavior: 把案例获奖、报告提交、政策建议、部门交流、宣传触达直接写成政策采纳、治理改善、制度有效或权益保障
  required_tools_or_checks:
    - live_project_pool_scan
    - source_link_recency_audit
    - policy_governance_claim_trace
    - evidence_matrix
```

## 八、对低摩擦入口的影响

用户不需要说“请启用 policy_governance_claim_trace”。只要出现“公共政策、社会治理、法治、政务服务、数字政府、智能治理、政策案例、政策建议、模拟政协、提案、法规解读、普法、基层治理、公共服务、权益保障、部门采纳、制度优化”等自然语言，AI 就应自动触发。

用户若说“真实项目不只是 AI 科研”，AI 必须把真实项目池理解为现实时间所有主要项目类型，而不是当前目录、上一轮主题或自己熟悉的 AI/科研材料。
