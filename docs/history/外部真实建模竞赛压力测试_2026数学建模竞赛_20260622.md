## 二、为什么选择这个样本

前几轮已覆盖 AI/数据竞赛、社会实践、创新创业、工程硬件、开源贡献和科研数值链，但还缺少一类高频真实项目：数学建模、预测、优化、评价和决策支持。

这类项目的典型风险不是“有没有做事”，而是：

1. 模型在赛题条件下成立，不等于现实情境中成立。
2. 目标函数最优，不等于公共利益、组织成本或使用者体验最优。
3. 拟合效果好，不等于可解释、可复现、可迁移。
4. 论文结构完整，不等于数据、假设和边界经得起追问。
5. 允许使用 AI 工具辅助后，模型来源、代码生成、引用、原创性和准确性责任更需要显性追踪。

## 三、本轮暴露的新协议缺口

现有 `numeric_calculation_trace` 能处理数字、公式、单位和复算问题，但不能覆盖“模型是否能支撑现实决策”。

例如一个队伍可能写出漂亮的县域教育资源配置模型：

1. 用学生成绩、通勤距离、师资数量和学校容量构建优化模型。
2. 得出若干资源配置建议。
3. 论文获得较好评价。

但如果没有说明数据来源、政策约束、学生家庭选择、教师流动、财政预算、模型敏感性和不确定性，就不能声称“已经找到县域教育资源配置的最优方案”。更稳妥的结论只能是“在给定数据和假设下得到一个可讨论的优化框架”。

## 五、最小结论门

建模类项目只有同时满足以下条件，才能把模型结果上升为较强建议：

1. 关键数据来源清楚，缺失、偏差和时间范围已说明。
2. 关键假设可列出，且知道哪些假设最脆弱。
3. 目标函数、约束和评价指标与真实问题的价值目标一致。
4. 至少有 baseline、历史回测、交叉验证、专家校验或现实案例对照之一。
5. 至少做过敏感性或鲁棒性检查。
6. 对 AI 工具参与生成的代码、文字、模型想法和引用做过责任确认。
7. 明确写出不能外推的场景。

如果上述条件不足，最终报告必须降级：

| 不能写 | 应降级为 |
| --- | --- |
| 模型证明该方案最优 | 在给定假设下，该模型给出一个候选方案 |
| 该策略可直接推广到现实 | 该策略需要结合当地约束和现场数据再验证 |
| 数据显示问题已经被解决 | 当前数据支持一个有限范围内的解释 |
| AI 生成的方案可靠 | AI 辅助产物已被人工校验到某一边界 |

## 六、对前置协议的修正

新增失败模式：

```yaml
agent_failure_modes:
  - failure_id: F43-01
    name: model_decision_gap
    symptom: 把模型跑通、论文完成、仿真结果、预测精度或竞赛获奖误当成现实决策成立
    root_cause: 只追踪数值和公式，没有追踪数据质量、假设、目标函数、验证、敏感性和现实转译
    trigger_signal:
      - 数学建模、预测、优化、评价体系、仿真或决策支持项目
      - 结论中出现“最优、证明、可推广、解决、应当采取”等强表述
      - 使用 AI 工具辅助生成模型、代码、文字或引用
    corrected_behavior: 启用 model_validity_trace，并把模型结论降级到证据能够支持的范围
    detection_check: 桌面协议应能检索到总规则 40、执行闸门 99、model_validity_trace 和 model_decision_gap
    regression_case: R43-model-validity-trace
    status: mitigated_with_trace_module
```

新增回归样例：

```yaml
agent_regression_cases:
  - case_id: R43-model-validity-trace
    user_prompt_pattern: 用户要求做数学建模、预测、评价、优化、仿真或政策/运营决策建议，并希望成果能参赛或对外汇报
    expected_behavior: 先核验官方规则或真实问题来源，再启用 model_validity_trace，区分模型结果、验证证据和现实可行动建议
    forbidden_behavior:
      - 只给模型框架和论文目录
      - 把拟合精度、仿真图或获奖可能性当作现实有效性
      - 不说明数据、假设、约束和不能外推范围
    required_tools_or_checks:
      - live_project_pool_scan
      - source_link_recency_audit
      - trace_module_router
      - model_validity_trace
    pass_condition: 最终表达中能清楚看到模型能支持什么、不能支持什么、需要什么证据才能升级结论
    last_tested: 2026-06-22
```

## 七、对用户低摩擦入口的影响

用户不需要说“model_validity_trace”。只要出现以下自然语言，AI 应自动触发：

1. “数模、数学建模、建模比赛、预测模型、评价体系、仿真、优化、路径规划、资源配置”。
2. “帮我做一个方案并证明最优/有效/可推广”。
3. “用模型支撑政策建议、运营建议、公共问题解决方案”。

用户可见回复只需要说明：

“这是建模/决策支持类项目，我会额外检查数据、假设、验证、敏感性和现实转译，避免模型结论外推。”
