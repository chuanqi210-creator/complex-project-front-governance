## 一、为什么选这个样本

本轮按用户修正后的范围执行：真实项目不只是本地 `ai 科研` 文件，也不只是 AI 科研项目，而是现实时间所有比赛项目和真实组织项目。科研诚信这一类适合作为新压力测试，是因为它横跨基金申请、论文发表、大创/挑战杯/国创赛、数模、课程研究、社会调查、实验报告、数据集、模型训练、伦理审批、同行评议和成果验收。

选择样本：

1. 国家自然科学基金委《案件通报（2026 年第 1 批）》：`https://www.nsfc.gov.cn/p1/2812/2816/99722.html`

## 二、暴露出的协议缺口

旧协议已经能处理数值计算、调研统计、团队署名、质量验收、数据隐私、知识产权和生命健康，但缺少一条把“研究真实性、可重复性、伦理合规和科研不端风险”合并管理的主 trace。

如果没有这一层，AI 很容易写出这些不可靠结论：

1. “论文已发表，所以研究结论可靠。”
2. “基金获批，所以研究方案和经费使用没有问题。”
3. “伦理批件编号存在，所以参与者权益已充分保护。”
4. “代码仓库已公开，所以实验可复现。”
5. “没有撤稿记录，所以不存在数据造假或图像处理问题。”
6. “获奖/结题/导师签字，所以数据真实、署名公平、过程合规。”

这些判断的共同问题是把外显凭证当成完整证据链。真正需要追踪的是研究方案、原始数据、实验记录、样本链、数据处理、图像原件、统计计划、代码环境、伦理审查、知情同意、作者贡献、利益冲突、同行评议边界、更正撤稿、独立复现、投诉调查和经费使用映射。

## 五、判断规则

1. 论文、基金、获奖、伦理批件、代码仓库、审稿通过和无撤稿记录只能说明“有外显材料”，不能直接证明数据真实、结论可重复、伦理充分或无不端。
2. 同行评议和导师签字是审核流程，不是复现实验、原始数据核验或伦理执行核验的替代物。
3. 伦理审批不能替代实际知情同意、退出权、样本使用范围、隐私保护和不良事件处置。
4. 代码公开不能自动等于可复现；必须记录环境、依赖、随机种子、数据版本、运行步骤、指标重算和失败复现。
5. 未发现撤稿或未被调查不能写成“无不端”；最多写成“截至当前来源未见公开撤稿/调查结论”。
6. 涉及作者和贡献时同步触发 `team_contribution_authorship_collaboration_trace`；涉及个人信息、样本或数据集时同步触发 `data_algorithm_privacy_governance_trace` 和 `stakeholder_consent_participant_protection_trace`；涉及健康主张时同步触发 `biomedical_health_claim_trace`。

## 六、迁移到真实项目的用法

| 项目类型 | 常见误判 | 最小补证 |
| --- | --- | --- |
| 大创/挑战杯/国创赛 | 获奖、结题、导师推荐等于研究可靠 | 方案、数据、实验记录、贡献署名、复现步骤、边界说明 |
| 数学建模/数据竞赛 | 论文、代码、榜单等于模型可复现 | 数据版本、指标重算、随机种子、环境、外推边界 |
| 社会调查/课程研究 | 问卷样本和统计显著等于总体事实 | 抽样框、问卷版本、清洗记录、访谈同意、统计计划 |
| 生医/心理/教育实验 | 伦理批件等于参与者保护充分 | 知情同意、退出机制、风险分层、数据安全、不良事件 |
| 论文/基金申请 | 发表或获批等于无不端 | 原始数据、图像原件、作者贡献、利益冲突、投诉调查 |
| AI/数据集/模型训练 | 开源仓库等于可复现和合规 | 数据来源、授权、环境、依赖、敏感数据边界、复现报告 |

## 九、最小结论门

没有以下材料之一时，不得写“科研诚信可靠、数据真实、伦理合规、结论可重复、无学术不端、经费合规”：

1. 研究方案或注册计划。
2. 原始数据、实验记录、样本链和时间戳。
3. 数据清洗、剔除、转换、版本历史。
4. 图像原件和处理记录。
5. 统计分析计划、检验选择、阴性结果和多重比较处理。
6. 代码环境、依赖、随机种子、复现步骤和重算记录。
7. 伦理审查、知情同意、退出权、隐私和生物安全边界。
8. 作者贡献、署名同意、利益冲突和经费来源。
9. 更正、撤稿、投诉、调查、复现失败或独立复现记录。

证据不足时使用替代表达：

1. “已形成阶段性研究材料。”
2. “目前只能证明完成了某种申报、发表、评审或结题流程。”
3. “科研诚信、伦理合规和可重复性仍需要补充原始证据链。”
4. “该结论不能外推为数据真实、无学术不端或长期可靠。”
