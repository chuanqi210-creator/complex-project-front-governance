# 外部真实法律实务压力测试：2026 北京市大学生模拟法庭竞赛

current_as_of：2026-06-22

本轮用途：继续修正“真实项目只等于 AI 科研或当前目录项目”的偏差，用现实时间的法律实务/模拟法庭项目压力测试前置治理协议。核心问题不是“法学项目是否重要”，而是防止把模拟训练、普法宣讲、法律文书或竞赛获奖写成真实法律服务、权益救济、胜诉结果、合规风险降低或法律职业能力已经形成。

## 2. 为什么需要新增法律实务 trace

旧协议已经有 `policy_governance_claim_trace`，能处理政策建议、法治宣传、制度优化、部门采纳等问题。但法律实务项目还多出一组更硬的边界：

1. 谁有资格、授权和责任去提供法律服务。
2. 是否存在真实当事人、真实事实、真实证据和程序状态。
3. 是否完成保密、隐私、利益冲突和职业伦理检查。
4. 结果是模拟辩论胜出、文书得分、普法触达，还是法院/仲裁/行政/调解程序中的真实结果。
5. 学生能力是一次竞赛表现，还是经多次反馈、实务监督和真实任务验证后的职业能力。

因此，法律项目不能只问“建议有没有道理”，还必须问“这个角色能不能做、有没有授权、程序到哪一步、结果能不能归因、风险由谁承担”。

## 6. 举例：如何把故事讲实

以下是举例，不是本轮真实发生事实。

一支法学院队伍参加模拟法庭，案情围绕平台用工劳动争议。最初报告写成：“团队以扎实法律论证维护了新就业形态劳动者权益，推动了基层劳动纠纷解决。”这句话的问题在于，模拟法庭案情不等于真实当事人案件，辩论胜出不等于劳动者获得赔偿或劳动关系被确认，书状评分也不等于法院、仲裁委或企业采纳。

按 `legal_practice_effect_trace` 重写后，故事可以变成：

1. 起点：团队在模拟法庭训练中发现，平台用工案件的事实认定、证据留存和权利救济路径，对普通劳动者并不直观。
2. 证据边界：竞赛只能证明团队完成了高强度法律检索、书状写作和口头辩论训练，不能证明真实纠纷已解决。
3. 转化动作：团队把竞赛中的争点拆成“劳动关系判断、证据保存、仲裁时效、平台规则阅读、求助渠道”五个普法模块，经指导教师和律师校核后，用于社区或校园劳动权益讲座。
4. 服务闭环：讲座现场不做个案代理，只提供法律信息和转介清单；遇到真实个案，记录当事人同意、隐私保护和转介去向，由法律援助机构、律师事务所或学校法律诊所按制度接续。
5. 可以声明的成果：团队形成了从模拟法庭训练到普法工具包的转化，提升了受众对法律路径和证据保存的理解，并建立了合规转介机制。
6. 不能声明的成果：不能写成团队已经胜诉、已经完成法律援助、已经解决劳动纠纷或已经降低某企业法律风险，除非有相应授权、程序和结果证据。

这个版本的故事性不是靠拔高，而是靠“我们原以为模拟赛讲清道理就足够，后来发现真实法律问题首先卡在授权、证据、程序和转介；于是把竞赛能力转化为更负责任的公共法律信息服务”。

## 7. 写回动作

1. 桌面协议总规则 51：法律实务、模拟法庭、法律援助、普法咨询、合规建议等项目必须维护 `legal_practice_effect_trace`。
2. 桌面协议执行闸门 110：检查法律项目是否区分模拟训练、法律教育、普法服务、真实个案、法律援助和执业监督。
3. 桌面协议失败模式 42：新增 `legal_reality_gap`。
4. 跨域路由表新增法律实务/模拟法庭/法律援助触发项。
5. 低摩擦入口新增法律实务类自然语言触发。
6. 现实时间项目渠道库新增北京市大学生模拟法庭竞赛样本，并强调真实项目池覆盖现实时间所有比赛和真实组织项目，不以 AI 科研为中心。
