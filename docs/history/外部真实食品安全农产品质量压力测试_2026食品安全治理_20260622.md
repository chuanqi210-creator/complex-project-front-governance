# 外部真实食品安全农产品质量压力测试：2026 食品安全治理

生成日期：2026-06-22

本文件用于把“食品安全、农产品质量、供应链追溯、学校食堂、直播助农、网络食品销售、消费者权益”等真实项目，转化为前置治理协议可触发、可审查、可回归的 trace。它不是食品安全报告，也不替代监管、检测或法律意见。

## 4. 对重大项目的改写要求

### 4.1 项目叙事降级

错误写法：

> 我们帮助当地农产品上架平台，产品有合格证和检测报告，直播销售反馈很好，因此证明该地农产品质量安全、品牌可信、可复制推广。

协议改写：

> 我们完成了农产品上架、直播销售和初步质量材料整理，但食品安全和农产品质量结论仍需绑定具体产品批次、检测项目、检验机构、证书覆盖范围、储运记录和不合格处置。当前更稳妥的结论是“形成了农产品质量材料和供应链追踪样表”，而不是“证明产品安全可靠或供应链闭环”。

### 4.2 学校食堂场景

如果项目涉及学校食堂、集中供餐、县中学生营养改善或校园食品安全，不能只看供应商资质和校方认可，还必须补齐：

1. 采购批次、配送批次和留样记录。
2. 原料来源、票证、承诺达标合格证和追溯码核验。
3. 贮存温度、冷链、加工流程和交叉污染风险。
4. 食品添加剂、农药兽药残留、微生物、过敏原和标签风险。
5. 不合格食品召回、停用、退货、复检和家校沟通流程。
6. 学生、家长、学校、供应商、监管部门之间的责任边界。

### 4.3 支教/社会实践场景

如果支队在县中或乡村实践中发现学生早餐、校园周边食品、食堂供应、助农产品或直播带货问题，可以把它转成一个公共议题，但必须保持边界：

```yaml
field_inquiry_template:
  question: 学生和老师最担心哪些食品安全问题？
  data_sources:
    - 学生匿名问卷
    - 教师和食堂管理者访谈
    - 校园周边食品店观察
    - 采购或供餐制度公开材料
    - 监管公开抽检信息
  cannot_do:
    - 不自行判断食品是否安全
    - 不公开点名未核实商家
    - 不诱导学生提供可识别健康经历
    - 不把少数访谈外推为全校食品安全状况
  useful_output:
    - 校园食品安全认知地图
    - 学生高频担忧清单
    - 食堂/周边/家庭食品安全信息来源差异
    - 面向学生的“如何读懂标签、抽检通报、合格证和追溯码”课程
```

## 6. 最小结论门

可以写：

1. “项目形成了食品安全/农产品质量认知课程、需求清单和信息核验工具。”
2. “项目梳理了公开抽检信息、合格证和追溯信息的阅读方法。”
3. “项目发现学生、教师、家长和供应商对食品安全信息的理解存在差异。”
4. “项目提出了待监管、学校或供应商进一步核验的风险点。”

不能写，除非证据链闭合：

1. “该产品安全可靠。”
2. “该供应链全程可追溯。”
3. “学校食堂食品安全风险已消除。”
4. “消费者权益已充分保障。”
5. “助农产品质量优良，可规模复制推广。”
