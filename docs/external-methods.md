# External Methods Mapped To Complex

Complex borrows method shapes, not dependencies. These references are used only when they improve the seven behavior-kernel actions.

## Problem Nature

- Cynefin helps separate obvious/complicated work from complex discovery work.
- Complex maps this to `project_nature`: `evidence_fill`, `model_discovery`, `mixed`, or `execution_delivery`.
- If the frame, model, metric, or explanation path is unsettled, protect discovery before evidence filling.

## Divergence And Convergence

- Double Diamond provides a simple rhythm: diverge before deciding, then converge before delivery.
- Complex uses this only for model-discovery or mixed work.
- Evidence-fill work should not pay unnecessary divergence overhead.

## Thought Search

- Tree of Thoughts and Graph of Thoughts show why single-path greedy reasoning can converge too early.
- Complex keeps candidate frameworks, rival positions, discriminating probes, and rollback routes when the project frame is not settled.
- Once convergence conditions are met, Complex switches back to evidence and delivery discipline.

## Argument Mapping

- IBIS-style issue / position / pro / con maps help define what the question actually is.
- Complex uses this in `templates/argument.md` and the model-discovery golden example.
- The point is not a large diagram; it is to stop a local evidence gap from hijacking an unsettled research frame.

## Agent Workflows

- Agent workflow patterns such as routing, orchestrator-workers, evaluator-optimizer, state persistence, interrupts, guardrails, handoffs, and tracing are useful engineering shapes.
- Complex maps them to decision rights, topology choice, transcript review, and recovery records.
- These are not required dependencies. Use the local tools and current environment first.

## Evaluation

- Agent evals shift attention from final answers to behavior traces: tool use, state recovery, decision quality, and user correction cost.
- Complex maps this to behavior cases, transcript review rules, filled examples, and `tools/verify_complex_integrity.py`.
- New rules should come from repeated real failures, not from naming another framework.
