# 端到端低摩擦入口模拟

current_as_of：2026-06-22

用途：测试用户只用自然语言提出重大项目时，AI 是否能自动启动前置治理、选择必要 trace、保持用户可见压缩，而不是要求用户理解协议术语。

## 一、模拟用户提示

```text
我们想做一个 AI + 县域教育相关的真实实践项目，也可能拿去参加比赛。
目标是做得有意义、能落地、最好能拿好成绩。
现在只有一些想法和零散材料。
请按前置治理协议推进，但不要让我填复杂表格。
```

## 二、AI 应识别出的项目类型

```yaml
project_type:
  - social_practice
  - education_public_issue
  - ai_product_or_tool_possibility
  - possible_competition_or_entrepreneurship
major_project_mode:
  trigger: public_impact + competition_goal + vague_materials + high_rework_risk
compressed_user_entry:
  status: triggered
```

## 三、trace_module_router 应选择的模块

只启用 3 个最高杠杆模块：

```yaml
trace_module_router:
  primary_claim_to_prove: 项目能对县域教育中的真实问题产生有证据的帮助，并具备比赛/展示价值
  triggered_modules:
    - module: live_project_pool_scan
      reason: 用户说真实实践项目、可能参赛，需要外部案例和当前项目渠道
    - module: service_impact_trace
      reason: 核心场景是县域教育和社会实践，必须避免把课程/活动当成影响
    - module: competition_to_deployment_trace
      reason: 涉及 AI 工具和比赛展示，必须区分 demo/评审表现与真实教育场景可用性
  explicitly_not_applicable:
    - module: prototype_validation_trace
      reason: 当前没有硬件/工程原型信号
    - module: open_source_contribution_trace
      reason: 当前没有上游开源贡献信号
  maybe_later:
    - business_model_validation_trace
    - numeric_calculation_trace
```

## 四、用户可见压缩回复样例

```text
我把它识别为“AI + 县域教育 + 社会实践/竞赛展示”的重大项目。
这一轮我先启用三个检查：现实项目渠道扫描、服务影响追踪、AI demo 到真实场景可用性的追踪。
我会先找同类真实比赛/实践案例，再把你们的想法压成一个可验证的项目抓手。
当前最大风险是：课程或 AI 展示看起来很漂亮，但无法证明对学生/老师真实有帮助。
下一步我会产出一份候选抓手池和最小验证闭环，不需要你填写治理表。
```

## 五、内部应写入的最小治理记录

```yaml
compressed_user_entry:
  user_prompt_is_sufficient: true
  questions_asked_to_user: 0
  assumption_recorded:
    - 以县域学生/教师真实需求为第一优先，而不是先做 AI 展示
    - 比赛成绩是外部约束，不是唯一目标
  next_action:
    - live_project_pool_scan
    - candidate_project_hook_pool
    - minimum_validation_loop
```

## 六、模拟结论

本次模拟通过：

1. 用户没有使用任何协议术语，仍可触发重大项目治理。
2. AI 只启用 3 个最高杠杆 trace，没有展示完整模板。
3. 用户可见回复在 5 项以内。
4. 复杂字段写入内部治理文件。
5. 下一步动作具体且不要求用户填表。

剩余风险：

1. 如果用户后续明确要做创业赛，应追加 `business_model_validation_trace`。
2. 如果项目包含量化成效、成本或问卷统计，应追加 `numeric_calculation_trace`。
3. 如果做成开源工具，应追加 `open_source_contribution_trace`。
